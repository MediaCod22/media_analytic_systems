# Open Science package: режимы аналитической работы с медиаданными

## Назначение репозитория

Этот пакет предназначен для сопровождения статьи «Режимы аналитической работы с медиаданными: от измерений к поддержке принятия решений» и обеспечивает воспроизводимое описание исследовательского дизайна, методологии PRISMA, структуры данных, кодировочной схемы и производных таблиц.

Исследование рассматривает трансформацию систем медиааналитики из инструментов мониторинга в инфраструктуру поддержки принятия решений (DSS, Decision Support Systems). Центральный результат — типология режимов работы с медиаданными и сравнительная оценка институционализации DSS-связи в российском, китайском и международном научных контурах.

## Исследовательская рамка

**Объект:** научные публикации о системах медиааналитики, медиамониторинга, медиаметрии, OSINT, public opinion monitoring / 舆情监测 и системах поддержки принятия решений.

**Предмет:** способы связи медиаданных с аналитическими продуктами и управленческими решениями.

**Цель:** предложить и обосновать типологию режимов работы с медиаданными, позволяющую перейти от описательного мониторинга к аналитике, поддерживающей принятие решений.

## Основные исследовательские вопросы

1. Какие режимы аналитической работы с медиаданными представлены в российском, международном и китайском научных контурах?
2. На каком уровне возникает разрыв между мониторингом медиасреды и поддержкой принятия решений?
3. Как различаются российский, международный и китайский контуры по частоте упоминания систем и явной DSS-связи?
4. Какой аналитический продукт производит каждый режим работы с медиаданными?

## Ключевые результаты

| Контур | Упоминание систем | DSS-связь | Динамика DSS-связи 2019–2024 |
|---|---:|---:|---:|
| Google Scholar | 16% | 13% | 8% → 14% |
| РИНЦ | 33% | 8% | 2% → 3% |
| CNKI | 45% | 35% | 28% → 37% |

Главный сравнительный результат: разрыв между российским и китайским контурами по DSS-связи составляет **4,4 раза**: 8% против 35%.

## Типология режимов аналитической работы с медиаданными

| Режим | Российский пример | Международный пример | Китайский пример | Основной результат |
|---|---|---|---|---|
| Медиаизмерения | Mediascope | Nielsen | — | Внимание, охват, контакт |
| Национальное медиапокрытие | Brand Analytics, SCAN | Factiva, LexisNexis | Юйцин-платформы | Наблюдение за медиасистемой, выявление нарративов |
| OSINT / data journalism | OSINT + BA/SCAN | GDELT + OSINT tools | Юйцин-анализ practices | Реконструкция событий, превращение данных в истории |
| Support / predictive analytics | Талисман, risk-аналитика | Signal AI, Dataminr | Сина юйцин / Wisers | Сценарии, предупреждение, рекомендация решения |
| AI-native layer | Исследовательские LLM pipelines | Dataminr, Signal AI | Midu / LLM reporting | Автоматизация извлечения сигнала |

## Как использовать

1. Разложить содержимое пакета в корень репозитория `MediaCod22/media_analytic_systems`.
2. Сохранить исходные выгрузки из Google Scholar, eLibrary/РИНЦ и CNKI в `data/raw/` или описать причины, по которым они не публикуются полностью.
3. Проверить производные таблицы командой:

```bash
python scripts/validate_tables.py
```

4. Создать GitHub Release с названием, например: `v1.0-open-science-companion`.
5. При подаче статьи указать репозиторий как открытое приложение к исследованию.
