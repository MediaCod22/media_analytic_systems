# Почему h-index платформ не используется в финальной версии статьи

## Постановка проблемы

В ранних версиях исследования применялся **индекс Хирша (h-index)** для программных платформ медиааналитики. Этот показатель был использован как эвристика «научной продуктивности» систем.

## Почему это методологически некорректно

| Аргумент | Пояснение |
|----------|-----------|
| **h-index измеряет авторов, не продукты** | Индекс Хирша разработан для оценки научной продуктивности **исследователей** — числа публикаций с числом цитирований ≥ порога. Программная платформа не является автором. |
| **Платформа ≠ аффилированная персона** | Публикации, упоминающие Brand Analytics или Mediascope, не «принадлежат» этим системам. Это сторонние исследователи, описывающие инструмент. |
| **Самоцитирование искажает метрику** | Техническая документация, white papers и маркетинговые материалы производителя искажают h-index, делая его непригодным для объективной оценки. |
| **Разные жизненные циклы** | Научный h-index предполагает стабильную карьеру автора. Программные продукты могут быть приобретены, ребрендированы или прекращены — их «научная биография» не сопоставима с человеческой. |
| **Платформы не публикуют статьи** | Платформа не может «опубликовать N статей с ≥N цитированиями». Публикации делают люди и организации, использующие платформу. |

## Что используется вместо h-index

В финальной версии исследования применяется ** Scientometric Visibility Index / Индекс научной представленности системы**:

```
SVI = (количество публикаций, упоминающих систему) / (максимальное количество упоминаний в данном контуре)
```

Это **нормализованная частота**, а не продуктивностная метрика. Она показывает, насколько часто система присутствует в научном дискурсе, не претендуя на оценку её «качества» или «мощности».

## Где сохранены первоначальные данные

Исходные библиометрические оценки (включая h-index) доступны в:
- `data/derived/legacy_bibliometric_notes.md` (если применимо)
- Исходные рабочие таблицы исследователя

Они **не участвуют в основных выводах статьи** и приведены только для документирования эволюции методологического дизайна.

## Вывод

Мы **осознанно отказались** от h-index как показателя, признав методологическую ошибку ранних версий. Это не «сокрытие» данных — это **прозрачность методологического процесса**.
