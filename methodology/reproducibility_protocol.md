# Протокол воспроизводимости

## Минимальный воспроизводимый пакет

Для проверки результатов исследования необходимы:

1. поисковые запросы по каждой базе данных;
2. таблица первичных результатов поиска или агрегированные значения;
3. критерии включения и исключения;
4. кодировочная книга;
5. кодировочная матрица или агрегированные производные таблицы;
6. описание процедуры расчета показателей;
7. описание ограничений доступа к коммерческим или закрытым базам.

## Расчет показателей

### Доля публикаций с упоминанием систем

```text
system_mention_share = publications_with_system_mentions / final_sample_size * 100
```

### Доля публикаций с DSS-связью

```text
dss_share = publications_with_dss_link / final_sample_size * 100
```

### Разрыв между Россией и Китаем по DSS-связи

```text
dss_gap_china_russia = 35 / 8 = 4.375 ≈ 4.4
```

## Проверка таблиц

В пакете есть скрипт `scripts/validate_tables.py`. Он проверяет:

- наличие ключевых CSV-файлов;
- соответствие итоговой выборки сумме трех контуров;
- корректность расчета разрыва 35 / 8;
- наличие пяти режимов аналитической работы.

## Требования к публикации данных

Если условия использования баз данных не позволяют публиковать полные выгрузки, рекомендуется публиковать:

- агрегированные таблицы;
- кодировочную схему;
- списки включенных публикаций с библиографическими метаданными;
- обезличенные/редуцированные таблицы без защищенных экспортных данных;
- скрипты проверки расчетов.

## Версионирование

Рекомендуется использовать GitHub Releases:

- `v0.1-draft` — черновой исследовательский пакет;
- `v1.0-submission` — версия, соответствующая поданной статье;
- `v1.1-revision` — версия после рецензентских правок;
- `v2.0-publication` — версия после публикации статьи.
